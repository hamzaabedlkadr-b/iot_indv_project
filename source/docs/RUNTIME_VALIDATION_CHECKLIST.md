# Runtime Validation Checklist

This checklist saves the remaining live tests for the day when the Heltec board is available.

## What Can Be Finished Without The Board

- firmware structure and configuration
- edge-side `MQTT` listener setup
- `TTN` payload decoder preparation
- measurement templates and analysis scripts
- final report structure and prompt-log templates

## What Still Needs The Board

- fresh serial log capture during runtime if a newer evidence bundle is needed
- optional repeat `MQTT/WiFi` or `LoRaWAN` publishing if configuration changes or newer screenshots are needed
- benchmark reruns if configuration changes
- final energy comparison measurements

## Home Preparation Checklist

Before reconnecting the board:

1. Confirm the broker host, port, and topic in `firmware/esp32_node/include/project_config.h`.
2. Choose the communication mode:
   `MQTT only`, `LoRaWAN only`, or `both`.
3. Install the edge listener dependency:
   `python -m pip install -r source\edge_server\mqtt_listener\requirements.txt`
4. Verify the local listener command you plan to use.
5. Prepare folders where logs, screenshots, and serial captures will be stored.

## Local MQTT Validation Checklist

Use this when the Heltec is available but `TTN` coverage is not required yet.

1. Start the `MQTT` broker.
2. Start the edge listener:
   `python source\edge_server\mqtt_listener\listen_aggregates.py --host <broker-ip> --port 1883 --topic project/adaptive-sampling-node/aggregate --csv source\edge_server\mqtt_listener\logs\aggregates.csv --jsonl source\edge_server\mqtt_listener\logs\aggregates.jsonl`
3. Flash the firmware with `MQTT` enabled.
4. Open the serial monitor and record:
   WiFi join,
   broker connection,
   aggregate creation,
   publish confirmation.
5. Let the board run long enough to capture several windows.
6. Save:
   serial output,
   listener terminal output,
   `CSV` and `JSONL` logs,
   screenshots if needed.
7. Run the analysis script:
   `python source\results\analyze_mqtt_results.py --input source\edge_server\mqtt_listener\logs\aggregates.csv --output-json source\results\summaries\mqtt_summary.json --output-md source\results\summaries\mqtt_summary.md`

## LoRaWAN Evidence Capture Checklist

Use this when the board is already running near a working `LoRaWAN` gateway and you want final screenshots for the submission.

1. Keep the same aggregate pipeline used for local `MQTT` validation.
2. Confirm the `TTN` credentials and radio configuration on the board.
3. Keep the payload format and decoder aligned with `cloud/ttn_payloads/`.
4. Open the serial monitor and wait for these lines together:
   `Prepared LoRaWAN aggregate payload`,
   `LoRaWAN heartbeat | ... joined=1`,
   `LoRaWAN uplink queued to radio`,
   `Event : Tx Done`.
5. Open the device page in `TTN` and capture:
   `Live data` with a fresh uplink,
   decoded payload values,
   device overview showing recent activity if available.
6. If `MQTT` is also enabled, capture both paths during the same run when possible.
